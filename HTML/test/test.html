Exemplar Template Engine : [!tEnMajor].[!tEnMinor].[!tEnPatch] - [!tEnCodename]

This is a showcase of all of the template engine's feature.

  • Constants are \[!likeThis], [!likeThis]!
  • Variables are \[@likeThis], [@likeThis]!
  • Files are \[#likeThis.html], [#likeThis.html]!
  • Tables are tags chained together are \[*fileName.html:@inputVariable], and then repeatedly call upon \[^] or \[^tablePosition].

Tables are a bit more involved, and work like this:

[*table.html:@tableInput]

As you can see, tables are a very powerful feature which open up many different possibilities.

Another little interesting factoid is that file tags are first tested relative to the template, then relative to the script, and finally absolute. This means that you can place your templates folder anywhere you damn well please, and have it work just fine. Some examples include [#relativeFile.html], [#test2/relativeFile.html], and [#test3/relativeFile.html].

Finally, this engine DOES support chaining together constant and variable tags with others. That can look a bit like \[#@likeThis], \[^!likeThis], or even \[@!likeThis]. You can see this in action with [!@thisLike].
